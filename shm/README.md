# Shared Memory (SHM) Visualization Project

이 프로젝트는 공유 메모리에 저장된 시스템 상태 정보(매체별 업무 장애 상태, 유량제어 등)를 사람이 읽기 좋은 형태로 출력하는 C 언어 예제와, 이를 데이터베이스(Oracle) 환경에서 재현하기 위한 SQL 스크립트를 포함하고 있습니다.

## 1. 프로젝트 개요
실제 운영 시스템의 공유 메모리 구조체(`HOST_INFO_SHM`)를 발췌하여, 특정 매체와 업무의 조합에 대한 장애 상태(`TaskHaltStat`)를 시각화하고 검증하는 기능을 제공합니다.

## 2. 주요 기능
- **공유 메모리 시각화**: 50x50 매트릭스 형태로 매체별 업무 장애 상태를 출력합니다.
- **장애 상태 필터링**: 장애가 발생한('1') 항목들만 모아서 매체별로 요약 출력합니다.
- **유량제어 확인**: 현재 유량제어가 적용 중인 인덱스를 식별하여 출력합니다.
- **단위 테스트**: `sub_CheckValue` 등 주요 로직에 대한 검증 기능을 포함합니다.
- **SQL 재현**: Oracle SQL을 통해 공유 메모리의 데이터 구조와 조회 로직을 동일하게 시뮬레이션할 수 있습니다.

## 3. 파일 구성
- `main.c`: 공유 메모리 구조체 정의 및 데이터 출력 로직 (C 소스)
- `Makefile`: 컴파일 및 실행 자동화 스크립트
- `shm.sql`: Oracle DDL/DML/SELECT 쿼리 예제 (일반 SQL)
- `shm.sqlnb`: 한글 주석이 포함된 SQL Notebook 파일
- `README.md`: 프로젝트 설명 문서 (본 파일)

## 6. 데이터베이스 연동 (Oracle)
`shm.sql` 또는 `shm.sqlnb`를 사용하여 공유 메모리 데이터를 데이터베이스 테이블에 정규화된 형태로 적재하고 관리할 수 있습니다.

### 데이터 구조 (정규화 버전)
- **ROW_KIND='M'**: 매체 마스터 정보 (매체 인덱스 및 한글명)
- **ROW_KIND='J'**: 업무 마스터 정보 (업무 인덱스 및 한글명)
- **ROW_KIND='H'**: 장애 상태 정보 (매체 인덱스, 업무 인덱스, 장애 여부)

### 주요 이점
- **가독성 향상**: 인덱스 번호가 아닌 실제 매체명과 업무명을 조인하여 상세한 장애 현황을 파악할 수 있으며, 필요에 따라 50자리 문자열 형태로도 자유롭게 변환하여 조회가 가능합니다.
- **유연한 쿼리**: 특정 업무에 대한 전체 매체의 장애 상태나, 특정 매체의 장애 통계를 쉽게 산출할 수 있습니다.
- **C 구조와의 일치**: `main.c`의 `TaskHaltStat[50][50]` 구조를 DB에서 가장 효율적으로 관리할 수 있는 형태로 재설계되었습니다.
