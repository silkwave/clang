# 통장 출력 문자열 생성기 (Bankbook String Generator)

이 프로젝트는 금융권의 레거시 통장 프린터 출력 방식에 맞춰 **120바이트 고정 폭(Fixed-width)**의 데이터 라인을 생성하는 C 언어 프로그램입니다.

## 🎯 프로젝트 목표
- **초보자 친화적 설계**: 포인터 연산(`*`, `++`) 대신 정수형 인덱스(`offset`)를 사용하여 버퍼 내 위치를 관리합니다.
- **레거시 대응**: 널 종료 문자(`\0`)가 없는 고정 폭 2차원 배열 구조를 지원합니다.
- **정교한 포맷팅**: 천 단위 쉼표와 소수점 처리가 포함된 금액 출력 로직을 구현합니다.

## ✨ 주요 기능
1. **인덱스 기반 오프셋 관리**: `offset = append_field(...)` 패턴을 사용하여 코드 가독성과 안정성을 높였습니다.
2. **금액 포맷팅 (`format_amount`)**:
   - 입력 문자열(예: "12345")을 천 단위 쉼표가 포함된 형식("12,345")으로 변환합니다.
   - 소수점이 `.00`일 경우 자동으로 제거하여 정수 형태로 표시합니다.
3. **필드별 정렬 지원**:
   - `append_char_field`: 문자 데이터를 **좌측 정렬**하여 추가합니다.
   - `append_amount_field`: 금액 데이터를 **우측 정렬**하여 추가합니다.
4. **안전한 문자열 처리**: `bounded_strlen`, `copy_bounded`를 통해 버퍼 오버플로우를 방지합니다.

## 📊 데이터 레이아웃 (120 Bytes)

하나의 통장 출력 라인은 다음과 같이 총 120바이트로 구성됩니다.

| 오프셋 | 필드명 | 폭 (Bytes) | 정렬 | 비고 |
|:---|:---|:---:|:---:|:---|
| 0 | 제어 문자 | 4 | - | `0xff 0x00 0x01 0x00` |
| 4 | 거래일자 | 10 | 좌측 | YYYYMMDD |
| 14 | 거래내용 | 20 | 좌측 | 한글/영문 혼용 |
| 34 | 출금금액 | 15 | 우측 | 천 단위 쉼표 포함 |
| 49 | 입금금액 | 15 | 우측 | 천 단위 쉼표 포함 |
| 64 | 현재잔액 | 20 | 우측 | 천 단위 쉼표/소수점 포함 |
| 84 | 예비 공간 | 36 | - | 공백(` `)으로 채움 |

## 🛠 주요 구조체
```c
/* 개별 거래 레코드 */
typedef struct {
    char trDt[9];     /* YYYYMMDD */
    char content[21]; /* 거래내용 */
    char outAmt[21];  /* 출금액 */
    char inAmt[21];   /* 입금액 */
    char balance[21]; /* 잔액 */
} BankbookRecord;

/* 최종 출력 버퍼 구조 */
typedef struct {
    char BnbkData[76][120]; /* 120바이트 고정 폭 * 76라인 */
} PRT_BNBK_MSG;
```

## 🚀 빌드 및 실행 방법

### 빌드
`Makefile`이 제공되므로 `make` 명령어로 간단히 빌드할 수 있습니다.
```bash
make
```

### 디버그 모드로 빌드
필드 추가 과정을 상세히 보려면 `DEBUG` 매크로를 활성화하여 빌드합니다.
```bash
make CFLAGS+='-DDEBUG=1'
```

### 실행
```bash
./bbnk
```

## ⚠️ 주의사항
- **인코딩**: 한글 처리 시 UTF-8 환경에서는 한글 1글자가 3바이트를 차지하지만 화면 폭은 2글자 폭을 차지하므로, 콘솔 출력 시 정렬이 어긋나 보일 수 있습니다. (실제 통장 프린터는 인코딩 방식에 따른 고정 폭 폰트를 사용합니다.)
- **부동 소수점**: 금액 처리에 `atof`를 사용하므로 극도로 정밀한 금융 계산이 필요한 경우 전용 자료형(Decimal) 사용을 권장합니다.
